**Текущий этап проекта: 5. Построить baseline-решение с минимальным функционалом**

3.3. Схема взаимодействия (Sequence Flow)
Пользователь -> Frontend: Заполняет форму.

Frontend -> Backend (API Gateway): POST /api/generate.

API Gateway -> Orchestrator: Передает данные (продукт, ЦА, тон).

Orchestrator -> NLP Service: Отправляет текст для извлечения интента.

NLP Service -> Orchestrator: Возвращает теги продукта (например: #еда #сыр #роскошь).

Orchestrator -> Template Service: Запрос шаблонов по тегам настроения (из формы) и семантическим тегам (из NLP).

Template Service -> Orchestrator: Список подходящих шаблонов (ID, URL картинки).

Orchestrator -> Generator Logic: Маппинг продукта на текст в меме.

Orchestrator -> Image Renderer: Для каждого шаблона отправляет (ID шаблона, готовый верхний/нижний текст).

Image Renderer -> Orchestrator: Ссылки на готовые изображения в хранилище.

Orchestrator -> DB: Сохраняет запрос в историю.

Orchestrator -> API Gateway -> Frontend: Отдает JSON с массивом ссылок на готовые мемы.

4. База данных мемов (Ключевой актив)
На этапе MVP мы не полагаемся на парсинг интернета в реальном времени. Мы создаем курируемую базу из 100-200 самых популярных и гибких шаблонов (форматов), которые можно адаптировать под рекламу.

Структура шаблона: ID, Название файла, Теги (1...n), Количество текстовых полей (1 или 2), Цвет текста по умолчанию, Позиция текста (координаты).

Пример шаблона: "Disaster Girl" (девочка улыбается на фоне пожара).

Теги: ирония, неожиданность, провал, последствия.

Применение для рекламы: Верхний текст: "Мой бюджет на контекст". Нижний текст: "Трафик, который я получил".

5. Технологический стек (Предложение)
Frontend: React + TypeScript.

Backend: Python (FastAPI / Django) — отличный выбор для интеграции NLP.

NLP: Библиотеки NLTK или spaCy для начального анализа, позже можно подключать Yandex GPT / ChatGPT API для более креативных текстов.

Инфраструктура: Docker, Docker Compose для локальной разработки.

6. Этапы развития после MVP
Версия 1.0 (запуск): Ручной выбор шаблонов по тегам, простой текстовый оверлей.

Версия 2.0: Интеграция с нейросетями (Kandinsky, Midjourney) для генерации уникальных мем-картинок, а не только использования шаблонов.

Версия 3.0: Рекламный кабинет — автоматический постинг в соцсети (VK, TG) с расписанием, A/B тестирование мемов.

🚀 Как запустить Baseline MVP
1. Клонировать репозиторий
git clone <repo_url>
cd DS_pj
2. Создать виртуальное окружение
python -m venv venv

Активировать:

Windows (PowerShell)

venv\Scripts\Activate.ps1

Mac / Linux

source venv/bin/activate
3. Установить зависимости
pip install -r requirements.txt
4. Настроить API ключ HuggingFace

Создать файл .env в корне проекта:

HF_API_KEY=your_huggingface_api_key

Получить ключ можно здесь:
https://huggingface.co/settings/tokens

Тип токена: Read

5. Запустить backend

В корне проекта выполнить:

uvicorn backend.main:app --reload

Сервис запустится по адресу:

http://127.0.0.1:8000
6. Открыть Swagger UI

Для тестирования API перейти в браузере:

http://127.0.0.1:8000/docs

Endpoint:

POST /generate

Пример запроса:

{
  "product": "бургер",
  "audience": "зумеры",
  "tone": "постирония"
}

Ответ:

{
  "top_text": "Когда запускаешь рекламу",
  "bottom_text": "И ждёшь клиентов",
  "image": "backend/generated/meme_xxxxx.png"
}

После выполнения запроса изображение мема сохраняется в папке:

backend/generated/
