## 2. Концепция продукта

### 2.1. Концепция и позиционирование
**Memetic** — это интеллектуальный инструмент для малого бизнеса, SMM-менеджеров и маркетологов, который позволяет адаптировать голос бренда под язык интернет-культуры.

Сервис автоматизирует процесс создания вирусных рекламных постов, соединяя семантику продукта с популярными мемными шаблонами, релевантными для конкретной целевой аудитории.

**Ключевая проблема, которую мы решаем:** создание качественного юмористического контента требует глубокого погружения в культуру мемов и занимает много времени. Наш сервис позволяет получить готовую креативную связку «продукт + мем» за 1 минуту.

### 2.2. База данных мемов (Ключевой актив)
На этапе MVP мы не полагаемся на парсинг интернета в реальном времени. Мы создаем **курируемую базу** из 100-200 самых популярных и гибких шаблонов (форматов), которые можно адаптировать под рекламу.


#### Пример шаблона: "Disaster Girl"
*Описание:* Девочка улыбается на фоне пожара.

- **Теги:** ирония, неожиданность, провал, последствия.


**Текущий этап проекта: 5. Построить baseline-решение с минимальным функционалом**

MVP реализует базовый pipeline генерации мемов с использованием LLM и шаблонов изображений.

## 🚀 Как запустить Baseline MVP

### 1. Клонировать репозиторий
```bash
git clone <repo_url>
cd DS_pj
```
### 2. Создать виртуальное окружение
```bash
python -m venv venv
```
Активировать окружение
Windows (PowerShell)
```bash
venv\Scripts\Activate.ps1
```
Mac / Linux
```bash
source venv/bin/activate
```
### 3. Установить зависимости
```bash
pip install -r requirements.txt
```
### 4. Настроить API ключ HuggingFace
Создать файл .env в корне проекта:
```bash
touch .env
```
или
```bash
nano .env
```
В открывшемся редакторе вставьте строку:
```bash
HF_API_KEY=your_huggingface_api_key
```

Получить ключ можно здесь:
https://huggingface.co/settings/tokens

Тип токена: Read

### 5. Запустить backend

В корне проекта выполнить:
```bash
uvicorn backend.main:app --reload
```
Сервис запустится по адресу:
http://127.0.0.1:8000/docs
### 6. Открыть Swagger UI

Для тестирования API перейти в браузере:
http://127.0.0.1:8000/docs

Endpoint:
POST /generate

Пример запроса:
```json
{
  "product": "бургер",
  "audience": "зумеры",
  "tone": "постирония"
}
```

Ответ:
```json
{
  "top_text": "Когда запускаешь рекламу",
  "bottom_text": "И ждёшь клиентов",
  "image": "backend/generated/meme_xxxxx.png"
}
```
После выполнения запроса изображение мема сохраняется в папке:
backend/generated/
