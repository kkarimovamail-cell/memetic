# Файл IMPROVEMENTS.md

## Улучшения сервиса Memetic по результатам обратной связи (Этап 7)

### 1. Введение

На этапе тестирования (14.03.2026) был проведён анализ текущего состояния MVP, результаты которого зафиксированы в файле [`TESTS.md`](TESTS.md). В ходе технического тестирования были выявлены критические недостатки, не позволяющие провести полноценное пользовательское тестирование. В период с 14 по 16 марта команда внедрила ряд улучшений, направленных на устранение этих проблем и приведение функциональности в соответствие с концепцией проекта.

### 2. Выявленные проблемы (из TESTS.md)

| ID  | Проблема                                  | Описание                                                                                                                                                      |
| --- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | **Одинаковые картинки**                   | При разных входных параметрах (product, audience, tone) сервис возвращал один и тот же мем‑шаблон. Отсутствовала база шаблонов и логика их выбора.          |
| 2   | **Одинаковый текст**                       | Генерируемый LLM текст (top_text / bottom_text) не зависел от входных данных – всегда возвращалась одна и та же фраза.                                      |
| 3   | **Пустые поля приводят к ошибке 500**      | Отсутствовала валидация входящих JSON – при передаче пустых строк сервер падал с внутренней ошибкой.                                                          |
| 4   | **Нет валидации тона**                     | Поле `tone` принимало произвольные значения, хотя для выбора шаблона необходим ограниченный перечень (например, funny, dark, dramatic).                     |

### 3. Внедрённые улучшения

#### 3.1. Реализация базы мем-шаблонов (ключевой актив)
- Создана курируемая база из 50+ шаблонов (планируется расширение до 100–200).
- Каждый шаблон описывается структурой:
  - `id` – уникальный идентификатор
  - `image_file` – имя файла с изображением
  - `tags` – список тегов (например, «ирония», «неожиданность», «провал»)
  - `text_fields` – количество текстовых полей (1 или 2)
  - `default_color` – цвет текста по умолчанию
  - `text_positions` – координаты наложения текста
- Добавлена логика выбора шаблона на основе тональности (`tone`) и аудитории (`audience`):
  - Для тона `funny` выбираются шаблоны с тегами «юмор», «ирония».
  - Для тона `dark` – шаблоны с тегами «сарказм», «безысходность».
  - Для тона `dramatic` – шаблоны с тегами «неожиданность», «пафос».
- Если точное совпадение по тегам не найдено, используется шаблон по умолчанию (Disaster Girl).

#### 3.2. Исправление генерации текста (LLM)
- Модифицирован промпт для HuggingFace API: теперь в шаблон запроса динамически подставляются значения `product`, `audience` и `tone`.
- Пример промпта:  
  *«Generate a funny two‑line meme text for a product '{product}' targeted at '{audience}' with a '{tone}' tone. Top line: …, bottom line: …»*
- Устранено кеширование ответов – каждый запрос отправляется к API с уникальными параметрами.
- Реализована генерация **трёх вариантов текста** для одного запроса, что позволяет пользователю выбрать наиболее подходящий.

#### 3.3. Добавлена валидация входных данных (Pydantic)
- Создана Pydantic‑схема `MemeRequest`, которая проверяет:
  - Наличие и непустоту полей `product`, `audience`, `tone`.
  - Допустимые значения для `tone` (один из: `funny`, `dark`, `dramatic`, `ironic`, `wholesome`).
  - Максимальную длину строк (продукт не более 100 символов и т.д.).
- При нарушении валидации сервис возвращает понятную ошибку 422 с описанием, вместо падения сервера (500).

#### 3.4. Улучшение интерфейса и обратной связи
- На фронтенд‑странице (http://127.0.0.1:8000) теперь отображаются три сгенерированных мема одновременно, что позволяет сравнить варианты.
- Добавлен индикатор загрузки во время обращения к API.

### 5. Заключение

Внесённые улучшения полностью устранили критические недостатки, зафиксированные в `TESTS.md`. Сервис теперь соответствует заявленной концепции: для каждого набора входных данных подбирается релевантный мем‑шаблон и генерируется уникальный текст.
